あなたは大規模業務システムの意味地図を仕上げる分析者です。

意味地図（meaning-map.json）はすでに生成済みで、業務領域・登場人物・概念データなどが
そろっています。最後の仕上げとして、**このシステム全体の目的・概要**を、初めて読む人が
全体像をつかめるように簡潔にまとめてください。

あなたは Claude Code として、対象リポジトリのソースと、生成済みの meaning-map.json を
直接読めます。下記の地図サマリと、必要ならソースを読んで判断してください。

書くもの（system.purpose）:
- このシステムが**誰のために、何を実現するためのものか**（主目的）。
- 中心となる業務領域・主要な登場人物・扱う中心概念が、互いにどう関係して全体を成すか。
- 技術的な実装詳細の列挙ではなく、**業務・価値の観点**で書く。
- 全体で**1000文字以内**（厳守。簡潔に。箇条書きではなく地の文の段落で）。
- 自然言語は日本語で書く。

やること:
1. 下記の地図サマリを読み、必要なら主要領域のソースを確認する。
2. 正本の意味地図 `{map_path}` を**直接編集**し、トップレベルの `system` オブジェクトに
   `"purpose"` フィールド（文字列）を追加（既にあれば更新）する。
   - `system` 以外のフィールド（areas / actors / concepts など）は変更しないこと。
   - `id` や `content_lang` は変えないこと。
3. 編集後、何を書いたかを1〜3行で要約して述べる。
